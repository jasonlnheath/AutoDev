## Mal Implementation Patterns

Special forms go in EVAL (check `isinstance(first, MalSymbol)`):
```python
if first.value == 'if':
    # (if condition then else)
    condition_expr, then_expr, else_expr = ast.items[1], ast.items[2], ast.items[3] if len(ast.items) > 3 else None
    condition = EVAL(condition_expr, env)
    # Only false and nil are falsy in Mal
    if isinstance(condition, (MalBoolean, MalNil)) and not (isinstance(condition, MalBoolean) and condition.value):
        if else_expr:
            return EVAL(else_expr, env)
        return NIL
    return EVAL(then_expr, env)

if first.value == 'fn':
    # (fn* params body...) - user-defined function
    params, *body_exprs = ast.items[1:]
    return MalFunction(params, body_exprs, env)

if first.value == 'do':
    # (do expr1 expr2 ... exprN) - evaluate each, return last
    result = NIL
    for expr in ast.items[1:]:
        result = EVAL(expr, env)
    return result
```
